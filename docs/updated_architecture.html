üé® Component Legend

üîµ CoordinatorAgent

Deterministic top-level orchestrator that loads the MDT roster, spawns CaseAgents, runs them, and aggregates results into the dashboard.

üü† CaseAgent

Per-patient pipeline built on ADK: runs a ParallelAgent of 5 specialist LlmAgents, then a CaseManager LlmAgent for synthesis.

üü£ ParallelAgent

Executes the 5 specialist agents concurrently for each patient to reduce latency.

üî∑ Specialist Agents (5 types)

LLM-powered agents: EHR, Pathology, Radiology, Genomics, Contraindication, each with its own custom tool and output_key.

üü¢ CaseManager

Final LlmAgent that reads all specialist outputs and produces a structured MDT readiness state for the patient.

üìä Data Flow Details

Phase 1: Parallel Data Gathering

All 5 specialist LlmAgents execute concurrently for each patient:

  • PathologyAgent ‚Üí Fetches tumour type, grade, ER/PR/HER2 status from SQLite.
  • EHRAgent ‚Üí Fetches demographics, comorbidities, allergies from JSON.
  • RadiologyAgent ‚Üí Fetches scan findings and flags unsigned reports from CSV.
  • GenomicsAgent ‚Üí Fetches mutations, CNAs, TMB, and HRD signals from JSON.
  • ContraindicationAgent ‚Üí Loads drug safety profiles and regimen rules from JSON.

Each specialist agent writes its summary into ADK session state using a unique output_key.

Phase 2: CaseManager Synthesis

CaseManager LlmAgent processes the combined state:

  • Reads all 5 output_key values from session state for the patient.
  • Constructs a comprehensive MDT readiness checklist.
  • Flags any BLOCKERS (e.g. unsigned MRI, missing genomic profiling).
  • Highlights key genomic findings (PIK3CA, HER2, ESR1, BRCA-related risk).
  • Cross-references contraindications with comorbidities to surface safety issues.
  • Outputs a final structured case state consumed by the coordinator for the dashboard.
